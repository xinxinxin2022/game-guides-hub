# 🎮 Game Guides Hub

A comprehensive English game guide website built with Astro, featuring detailed walkthroughs, builds, and collection guides for popular games.

## 📚 Featured Games

### ⚔️ Elden Ring
- Malenia Boss Guide (complete moveset breakdown & strategies)
- Starting Classes Ranked (tier list & recommendations)
- Limgrave Early Game Guide (step-by-step route)

### 🗼 Tower of Fantasy
- Navia Collection Guide (coordinates & puzzle solutions)
- Source Mechanics Guide (elements & combat rotation)
- Team Compositions 2026 (meta builds & strategies)

### 🌃 Cyberpunk 2077
- Cyber Psycho Build Guide (complete build with cyberware & perks)
- All Endings Explained (unlock requirements & consequences)
- Iconic Weapons Guide (top 10 weapons & locations)

## 🚀 Quick Start

```bash
# Install dependencies
npm install

# Start development server
npm run dev

# Build for production
npm run build

# Preview production build
npm run preview
```

## 🛠️ Tech Stack

- **Framework:** Astro 7.1.1
- **Styling:** Custom CSS (responsive, mobile-first)
- **Build:** Static site generation
- **Features:** SEO optimized, mobile-responsive, fast loading

## 📁 Project Structure

Inside of your Astro project, you'll see the following folders and files:

```text
/
├── public/
├── src/
│   └── pages/
│       └── index.astro
└── package.json
```

Astro looks for `.astro` or `.md` files in the `src/pages/` directory. Each page is exposed as a route based on its file name.

There's nothing special about `src/components/`, but that's where we like to put any Astro/React/Vue/Svelte/Preact components.

Any static assets, like images, can be placed in the `public/` directory.

## 🧞 Commands

All commands are run from the root of the project, from a terminal:

| Command                   | Action                                           |
| :------------------------ | :----------------------------------------------- |
| `npm install`             | Installs dependencies                            |
| `npm run dev`             | Starts local dev server at `localhost:4321`      |
| `npm run build`           | Build your production site to `./dist/`          |
| `npm run preview`         | Preview your build locally, before deploying     |
| `npm run astro ...`       | Run CLI commands like `astro add`, `astro check` |
| `npm run astro -- --help` | Get help using the Astro CLI                     |
